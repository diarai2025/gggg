# Настройка валидации форм

## Установка зависимостей

Для работы валидации форм необходимо установить следующие пакеты:

```bash
npm install zod @hookform/resolvers
```

## Что было реализовано

### 1. Схемы валидации (src/lib/validations.ts)

Созданы схемы валидации с использованием Zod для:
- **Кампании** (`campaignSchema`) - название, платформы, бюджет, телефон, локация
- **Лиды** (`leadSchema`) - имя, email, телефон, компания, статус, источник, заметки
- **Сделки** (`dealSchema`) - название, сумма, этап, вероятность, дата закрытия, заметки
- **Задачи** (`taskSchema`) - название, описание, статус, приоритет, дата выполнения
- **Логин/Регистрация** (`loginSchema`) - email, пароль, согласие
- **Восстановление пароля** (`resetPasswordSchema`) - email

### 2. Интеграция с react-hook-form

В компоненте `AIAdvertising`:
- ✅ Форма создания кампании использует `useForm` с `zodResolver`
- ✅ Форма редактирования кампании использует `useForm` с `zodResolver`
- ✅ Валидация происходит в реальном времени (`mode: 'onChange'`)
- ✅ Ошибки валидации отображаются под каждым полем
- ✅ Отправка формы блокируется при наличии ошибок
- ✅ Визуальная индикация ошибок (красная рамка полей)

### 3. Особенности валидации

- **Название кампании**: минимум 3 символа, максимум 100
- **Платформы**: минимум 1 выбрана
- **Бюджет**: положительное число, минимум ₸1,000
- **Телефон**: минимум 10 цифр, проверка формата
- **Локация**: опционально, максимум 200 символов

## Использование

После установки зависимостей валидация будет работать автоматически:

1. При вводе данных в форму происходит валидация в реальном времени
2. Ошибки отображаются под полями с красным цветом
3. Кнопка отправки блокируется при наличии ошибок
4. Форма не отправится, пока все поля не будут валидны

## Следующие шаги

Для полной интеграции валидации во все формы приложения:

1. Интегрировать валидацию в компонент `CRM` (лиды, сделки, задачи)
2. Интегрировать валидацию в компонент `Login`
3. Добавить валидацию в другие формы при необходимости
